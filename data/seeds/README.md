# `data/seeds/` — checked-in ingestion inputs

Pinakes builds its index from live data sources (CrossRef, OpenAlex, RSS, scrapers). For two journals/publishers, intermediate JSON files were generated once and committed so the canonical record of what was ingested travels with the repo. They are inputs to one-off ingestion scripts at the project root, not to the live web app.

| File | Size | First added | Producer | Consumer |
|---|---:|---|---|---|
| `peer_review_references.json` | 76 KB | 2026-03-31 | Hand-curated from issue 1.1 of *The Peer Review* | [`ingest_peer_review_1_1.py`](../../ingest_peer_review_1_1.py) |
| `ccdp_scraped.json` | 78 KB | 2026-04-07 | [`scrape_ccdp.py`](../../scrape_ccdp.py) (Computers and Composition Digital Press catalogue scraper) | Reviewed by hand, then ingested via the same script's load step |

## Re-running

- **`peer_review_references.json`** is hand-curated. The script `ingest_peer_review_1_1.py` reads it as-is; there is no producer to re-run. Edit the JSON directly if a reference needs to be corrected, then re-run the ingest script (which is idempotent — it uses INSERT-OR-IGNORE on `(source_article_id, target_doi)`).
- **`ccdp_scraped.json`** can be regenerated by re-running `python scrape_ccdp.py` from the repo root. The scraper is rate-limited (5 s between requests) and respects CCDP's robots.txt. Output is written back to this same path.

## Why these files are committed

These are small (under 100 KB each), human-readable, and form part of the bibliographic provenance: anyone who wants to audit which references the index attributes to *Peer Review* 1.1 can read [peer_review_references.json](peer_review_references.json) directly. Treating them as repository data rather than build artefacts is intentional.

## What does NOT belong here

- Output databases (`articles.db`, `*.sqlite`) — gitignored, lives on the Fly volume in production.
- Bulk fetch responses from CrossRef / OpenAlex — fetched at runtime by `fetcher.py` / `enrich_openalex.py` and never written to the repo.
- Anything containing PII, API keys, or non-public material.
