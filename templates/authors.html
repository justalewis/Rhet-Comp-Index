Methodology

Data source

Author names are extracted from CrossRef metadata for each article in the index. CrossRef stores author names in structured form (given name + family name), which Pinakes combines into a display name. Articles indexed via RSS feeds or web scraping may include authors extracted from different formats and are more prone to inconsistencies.

Name handling

Author names are stored as-is from CrossRef, with minimal normalization (whitespace trimming, consistent capitalization). No name disambiguation is performed: two different scholars with the same name are treated as one person, and the same scholar with variant name forms (e.g., “James Paul Gee” vs. “J. P. Gee”) may appear as separate entries. This is a known limitation of string-based author identification. Where available, ORCID identifiers from CrossRef are stored and displayed on author profile pages.

Author profiles

Each author profile page shows all articles in the index attributed to that name, along with co-author relationships, publication timeline, topic distribution, and institutional affiliation (from OpenAlex where available). Citation counts on author profiles reflect internal citations within this index only — not broader citation counts from Google Scholar or Web of Science.

Limitations

The index depends entirely on publisher metadata quality. Journals that deposit incomplete author data with CrossRef produce incomplete records here. The author count includes every unique name string, so it overstates the number of real individuals when name variants exist and understates it when different people share a name. Authors who publish exclusively in journals not indexed here will not appear at all.
